# Implementation Plan: XLMPredict Backend

## Overview

Xây dựng Express.js backend server trong thư mục `server/` theo nguyên tắc **"Contract-First, DB-as-Cache"**: smart contract Soroban là source of truth, PostgreSQL là read-optimized cache. Các tasks được sắp xếp theo thứ tự dependency: setup → core utilities → database → API routes → cron automation → frontend integration.

Tech stack: Express.js + TypeScript, pg (node-postgres), node-cron, zod, pino, vitest, fast-check.

## Notes

- Tasks đánh dấu `*` là optional — có thể skip để MVP nhanh hơn, nhưng nên implement để đảm bảo correctness
- Thứ tự dependency: Task 1 → 2 → 3 → 4 → 5 → 6 → 7 (checkpoint) → 8 → 9 → 10 → 11 → 12 → 13 → 14 → 15 (checkpoint) → 16 → 17
- **Contract-First principle**: Mọi state change đều phải qua smart contract trước, DB chỉ là mirror
- **Admin key security**: `ADMIN_SECRET_KEY` chỉ dùng trong `contractService.ts`, không bao giờ log hay expose trong response
- **Idempotency**: Tất cả sync/settle/cancel operations phải có idempotency check
- **BigInt handling**: pg trả về BIGINT dưới dạng string — cần parse với `BigInt()` trong tất cả DB queries
- Property tests (P1–P13) validate các invariants phổ quát, unit tests validate các examples cụ thể — cả hai đều cần thiết
- Integration tests cần test database riêng (Neon branch hoặc local PostgreSQL) — không dùng production DB
